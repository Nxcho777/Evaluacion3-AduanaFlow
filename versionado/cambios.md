# Registro de Control de Versiones - AduanaFlow

## Herramienta utilizada

GitHub.

## Objetivo

Registrar las iteraciones realizadas durante la construcción del prototipo, documentación de pruebas y organización de entregables de la Evaluación Formativa N°3.

## Registro de cambios

### Iteración 1: Creación de estructura base del repositorio

**Descripción:** Se creó la estructura inicial del repositorio con carpetas para prototipo, pruebas, versionado, coevaluación y evidencias.  
**Justificación:** Permite ordenar los entregables y mantener trazabilidad de la documentación.

### Iteración 2: Incorporación del enlace al prototipo

**Descripción:** Se agregó el enlace del prototipo funcional realizado en Balsamiq.  
**Justificación:** Permite acceder al prototipo navegable desde el repositorio.

### Iteración 3: Desarrollo del plan de pruebas ISO/IEC 25000

**Descripción:** Se documentaron criterios de calidad como eficiencia, fiabilidad, mantenibilidad y portabilidad.  
**Justificación:** Permite relacionar el prototipo con estándares de calidad de software.

### Iteración 4: Incorporación de casos de prueba

**Descripción:** Se agregaron casos de prueba asociados a los requisitos funcionales y no funcionales del sistema.  
**Justificación:** Permite validar el comportamiento esperado del prototipo.

### Iteración 5: Actualización final de documentación

**Descripción:** Se revisó la documentación general y se completaron secciones pendientes.  
**Justificación:** Permite entregar un repositorio claro, ordenado y alineado con los requerimientos de la evaluación.

## Ejemplos de mensajes de commit sugeridos

- Inicialización del repositorio con estructura base
- Agregado enlace del prototipo Balsamiq
- Incorporado plan de pruebas ISO25000
- Agregados casos de prueba del sistema
- Actualizada documentación final del proyecto
